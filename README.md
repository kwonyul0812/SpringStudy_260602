\## day 1

\### 미션 2

답 : 	@SpringBootConfiguration - 이 클래스는 설정 클래스임을 표시. 내부에 @Bean 정의를 가능하게 함.
@EnableAutoConfiguration - 프로젝트에 추가된 라이브러리를 보고, 필요한 설정을 자동으로 해줌.
@ComponentScan - 이 클래스가 있는 패키지와 하위 패키지 스캔 후 @Componenet, @Service, @Controller, @Repository 같은게 붙은 클래스를 찾아 자동으로 @Bean 생성

<br>

\### 회고

1. Spring Framework와 Spring Boot의 관계를 동기에게 한 문장으로 설명한다면? :  Spring Framework를 더 쓰기 쉽게 자동설정과 내장서버(톰캣)을 얹어준것.
2. Web Server와 WAS의 차이를 비유로 설명한다면? : Web Server는 정적인 파일을 보내주는 서버, Was는 자바 코드를 실행해 파일을 동적으로 실행 후 전달해주는 서버
3. 임베디드 WAS의 장점은 무엇이라고 생각하는가? : Tomcat 같은 서버가 jar 파일 안에 내장되어 있어 서버를 따로 설치할 필요없이 jar만 실행하면 됨. 간단함.
4. 오늘 작성한 코드 중 가장 이해가 안 되는 부분 1개와 그 이유는? : 제네릭 공부도중 와일드카드 부분이 이해하기가 어려웠음. 특히 와일드카드시 가져오기 혹은 사용하는 부분이 제한되어있는점.
5. Optional을 안 쓰고 null만 다루는 코드의 단점은? : null 체크를 하지 않으면 NullPointerException 오류가 발생할 수 있음.

\## Day 2

\###회고

1\. "IoC"를 동기에게 30초 안에 설명한다면? : IOC란 제어의 역전으로 객체의 생성, 조립, 생명주기 관리 등을 전부 스프링 컨테이너에게 넘겨주어 신경쓰지 않고도 자동으로 처리해주는것.

2\. 필드 주입(@Autowired)이 편한데도 생성자 주입을 권장하는 이유는? : final 키워드로 객체의 불변성 보장, 순환참조 방지 가능, 인자가 빠지면 컴파일러 에러 조기발견 가능, 테스트 용이

3\. Lombok이 없었다면 오늘 작성한 DTO 코드는 몇 줄이 더 늘었을까? : 수십줄은 더 늘었을것.

4\. DTO와 Entity를 같이 쓰면 어떤 문제가 생긴다고 했는가? : Entity를 DTO처럼 같이 쓰면 컬럼명이 외부에 노출, 비밀번호와 같은 내부필드 포함 위험, jpa 지연로딩 객체가 직렬화될때 LazyInitializationException 오류 발생, DB 스키마 바꾸면 API도 깨짐

5\. 순환참조 에러가 났을 때 무엇을 느꼈는가? 코드 설계에서 무엇을 조심해야 할까? : 코드 설계에서 서로가 서로에게 의존성 주입 (DI) 하지 않도록 구현시 주의해야함.
